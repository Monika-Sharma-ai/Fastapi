⚡ What is FastAPI?

FastAPI is a modern Python web framework for building APIs quickly and efficiently. It's designed to be fast (on par with Node.js and Go), easy to use, and produce production-ready code with minimal effort.

Key Features of FastAPI

  • 🚀 Speed: One of the fastest Python frameworks (comparable to NodeJS and Go)
  • ⚡ Easy: Intuitive syntax, easy to learn and use
  • 📚 Auto Documentation: Automatic interactive API docs (Swagger UI, ReDoc)
  • 🔍 Type Hints: Full support for Python type hints
  • ✅ Data Validation: Built-in validation using Pydantic
  • 🔐 Security: OAuth2, JWT, and API key support built-in
  • ⚙️ Modern Python: Uses async/await for high performance
  • 🧪 Testing Friendly: Easy to test with provided TestClient

Built on Starlette & Pydantic

FastAPI is built on top of two powerful libraries:

🌟 Starlette

A lightweight, high-performance ASGI web framework. Provides the core web functionality that FastAPI uses.

🔍 Pydantic

A data validation library using Python type hints. Automatically validates request data and converts it to Python objects.

FastAPI Simplicity: Hello World

Creating your first API endpoint is incredibly simple:

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
  return {"message": "Hello, World!"}

That's it! This creates a working API with:

  • ✅ A GET endpoint at `/`
  • ✅ Returns JSON response
  • ✅ Automatic documentation at `/docs`
  • ✅ Type safe

🛣️ Routes and Endpoints: Building Your API

A route is a URL path in your API. An endpoint is a route combined with an HTTP method (GET, POST, etc.).

Basic Route Syntax

🎯 Simple GET Endpoint

from fastapi import FastAPI

app = FastAPI()

@app.get("/items")
def get_items():
  return {"items": ["item1", "item2", "item3"]}

Breaking it down:

  • @app.get("/items") - Decorator: HTTP GET method on /items route
  • def get_items(): - Function that handles the request
  • return {...} - Automatically converted to JSON response

All HTTP Methods in FastAPI

# GET - Retrieve data
@app.get("/items")
def read_items(): ...

# POST - Create data
@app.post("/items")
def create_item(): ...

# PUT - Replace entire resource
@app.put("/items/{item_id}")
def update_item(item_id: int): ...

# DELETE - Remove data
@app.delete("/items/{item_id}")
def delete_item(item_id: int): ...

# PATCH - Partial update
@app.patch("/items/{item_id}")
def patch_item(item_id: int): ...

# HEAD - Like GET but no body
@app.head("/items")
def head_items(): ...

# OPTIONS - Describe options
@app.options("/items")
def options_items(): ...

Path Parameters: Dynamic Routes

Use curly braces {param} to create dynamic routes:

📍 Basic Path Parameter

@app.get("/users/{user_id}")
def get_user(user_id: int):
  return {"user_id": user_id, "name": "John Doe"}

What happens:

  • Request: GET /users/123
  • Parameter: user_id = 123
  • Type validation: Must be an integer, FastAPI converts automatically
  • Response: {"user_id": 123, "name": "John Doe"}

📍 Multiple Path Parameters

@app.get("/users/{user_id}/posts/{post_id}")
def get_user_post(user_id: int, post_id: int):
  return {
    "user_id": user_id,
    "post_id": post_id,
    "content": "This is a sample post"
  }

Request: GET /users/123/posts/456

📍 String Path Parameters

@app.get("/files/{file_path}")
def read_file(file_path: str):
  return {"file_path": file_path}

Request: GET /files/documents/report.pdf

Note: String parameters accept slashes for file paths

Query Parameters: Filtering & Options

Query parameters are optional filters passed in the URL after ?:

❓ Basic Query Parameter

@app.get("/items")
def get_items(skip: int = 0, limit: int = 10):
  return {
    "skip": skip,
    "limit": limit,
    "items": ["item1", "item2", ...]
  }

Examples:

  • GET /items → skip=0, limit=10 (defaults)
  • GET /items?skip=5 → skip=5, limit=10
  • GET /items?skip=5&limit=20 → skip=5, limit=20

❓ Required Query Parameter

@app.get("/search")
def search(q: str, category: str = "all"):
  return {
    "query": q,
    "category": category,
    "results": [...]
  }

Examples:

  • GET /search?q=python → works
  • GET /search → ERROR (q is required)
  • GET /search?q=python&category=courses → works

❓ Query Parameter with Type Validation

@app.get("/products")
def get_products(skip: int = 0, limit: int = 10, in_stock: bool = True):
  return {
    "skip": skip,
    "limit": limit,
    "in_stock": in_stock
  }

Examples:

  • GET /products?in_stock=true → in_stock = True
  • GET /products?in_stock=false → in_stock = False
  • GET /products?skip=10&limit=5&in_stock=true → all work

Request Body: Sending Data

For POST, PUT, PATCH requests, you send data in the request body. Use Pydantic models:

📤 Creating a Pydantic Model

from fastapi import FastAPI
from pydantic import BaseModel

class Item(BaseModel):
  name: str
  price: float
  description: str = "No description" # Optional
  is_available: bool = True

@app.post("/items")
def create_item(item: Item):
  return {
    "message": "Item created",
    "item": item
  }

Request Body (JSON):

{
  "name": "Laptop",
  "price": 999.99,
  "description": "High-performance laptop"
}

Response:

{
  "message": "Item created",
  "item": {
    "name": "Laptop",
    "price": 999.99,
    "description": "High-performance laptop",
    "is_available": true
  }
}
Automatic Validation: FastAPI automatically validates that the JSON matches the model. If a required field is missing or has wrong type, it returns a 422 error!

Combining Path, Query, and Body Parameters

🔀 All Together

from pydantic import BaseModel

class UpdateItem(BaseModel):
  name: str
  price: float

@app.put("/items/{item_id}")
def update_item(item_id: int, item: UpdateItem, q: str = None):
  return {
    "item_id": item_id,
    "item": item,
    "query": q
  }

Example Request:

  • Path: /items/42
  • Query: ?q=update_now
  • Body: {"name": "New Name", "price": 19.99}

▶️ Running with Uvicorn: Starting Your Server

Uvicorn is an ASGI server that runs your FastAPI application. It handles incoming HTTP requests and passes them to your FastAPI app.

What is ASGI?

ASGI (Asynchronous Server Gateway Interface) is a specification for asynchronous Python web servers and applications. It allows FastAPI to handle many concurrent requests efficiently.

Step 1: Create Your FastAPI App

Create a file named main.py:

from fastapi import FastAPI

app = FastAPI()

@app.get("/")
def read_root():
  return {"message": "Hello, World!"}

@app.get("/items/{item_id}")
def read_item(item_id: int, q: str = None):
  return {"item_id": item_id, "q": q}

Step 2: Run with Uvicorn

⚙️ Basic Command

uvicorn main:app --reload

Breaking it down:

  • main - File name (main.py)
  • app - Variable name containing FastAPI instance
  • --reload - Auto-reload on code changes (development only)

Running Successfully: What You'll See

INFO:     Uvicorn running on http://127.0.0.1:8000 (Press CTRL+C to quit)
INFO:     Uvicorn server process started
INFO: Started server process [1234]
INFO: Waiting for application startup.
INFO: Application startup complete.

Your API is now running at http://127.0.0.1:8000 (localhost:8000)

Uvicorn Configuration Options

🔧 Specify Host and Port

# Default is localhost:8000
# To listen on all network interfaces:
uvicorn main:app --host 0.0.0.0 --port 8000

# Use different port
uvicorn main:app --port 3000

# Use different host
uvicorn main:app --host 192.168.1.5

📊 Workers (Production)

# Single worker (default, development)
uvicorn main:app

# Multiple workers (production, use 4x CPU cores)
uvicorn main:app --workers 4

🔄 Auto-reload (Development)

# Watch for changes and auto-reload
uvicorn main:app --reload

# Development preset (includes reload)
uvicorn main:app --env-file .env --reload

🔐 SSL/HTTPS (Production)

# With SSL certificates
uvicorn main:app --ssl-keyfile=key.pem --ssl-certfile=cert.pem

Common Uvicorn Options Summary

Option Default Purpose Example
--host 127.0.0.1 Server host/IP --host 0.0.0.0
--port 8000 Server port --port 3000
--reload False Auto-reload on code changes --reload
--workers 1 Number of worker processes --workers 4
--log-level info Logging level --log-level debug
--ssl-keyfile None SSL key file path --ssl-keyfile key.pem

Testing Your Running API

Once the server is running, test it:

🧪 Using curl (Command Line)

curl http://127.0.0.1:8000/

curl "http://127.0.0.1:8000/items/42?q=python"

🧪 Using Python requests

import requests

response = requests.get("http://127.0.0.1:8000/")
print(response.json()) # {"message": "Hello, World!"}

response = requests.get("http://127.0.0.1:8000/items/42?q=python")
print(response.json()) # {"item_id": 42, "q": "python"}

Press CTRL+C in your terminal to stop the Uvicorn server.

📖 Auto-Generated API Documentation

One of FastAPI's superpowers is automatic generation of interactive API documentation. This is generated from your code using OpenAPI specifications.

Two Documentation Interfaces

🎨 Swagger UI

A modern, interactive interface for testing your API.

URL: http://localhost:8000/docs

Features:

  • Interactive endpoint exploration
  • Try-it-out button to test endpoints
  • View request/response examples
  • See parameter documentation
  • Automatic request body examples

📑 ReDoc

A clean, alternative documentation interface optimized for reading.

URL: http://localhost:8000/redoc

Features:

  • Organized documentation hierarchy
  • Easy searching
  • Detailed schema information
  • Read-only (no testing directly)

What Gets Documented Automatically

FastAPI uses your Python function signatures and type hints to generate documentation:

📝 Example Code

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="My API", description="A simple API")

class User(BaseModel):
  """User model"""
  id: int
  name: str
  email: str
  age: int = None

@app.get("/users/{user_id}", response_model=User)
def get_user(user_id: int):
  """Get a user by ID"""
  return {"id": user_id, "name": "John", "email": "john@example.com"}

Automatically Documented:

  • Endpoint path: /users/{user_id}
  • HTTP method: GET
  • Path parameter: user_id (integer)
  • Response model: User
  • Docstring: "Get a user by ID"
  • All model fields with types

📝 Add Docstrings and Examples

from fastapi import FastAPI
from pydantic import BaseModel

class Item(BaseModel):
  """Product item"""
  name: str
  price: float
  description: str = None
  
  class Config:
    json_schema_extra = {
      "example": {
        "name": "Laptop",
        "price": 999.99,
        "description": "High-performance laptop"
      }
    }

@app.post("/items", response_model=Item)
def create_item(item: Item):
  """
  Create a new item.
  
  - **name**: Name of the item (required)
  - **price**: Price in USD (required)
  - **description**: Optional description
  """
  return item

Disabling Documentation

For production, you might want to disable documentation:

🔒 Disable All Docs

from fastapi import FastAPI

app = FastAPI(docs_url=None, redoc_url=None, openapi_url=None)

🔒 Custom URLs

app = FastAPI(
  docs_url="/api-docs", # Custom Swagger URL
  redoc_url="/api-reference", # Custom ReDoc URL
  openapi_url="/api/openapi.json" # Custom OpenAPI URL
)

Using Documentation in Development

💡 Best Practices

  • Keep documentation enabled during development
  • Use Swagger UI (/docs) to test endpoints interactively
  • Add meaningful docstrings to functions
  • Include examples in Pydantic models
  • Document error responses with status codes
  • Share ReDoc link with stakeholders for review

⚠️ Security Note

In production, consider disabling documentation or protecting it with authentication to avoid exposing your API structure to potential attackers.
